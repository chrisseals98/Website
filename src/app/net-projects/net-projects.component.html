DOTNET Projects


I used the .NET 6.0 framework for some projects during college for my CSCI-2910 class. This class was about server-side programming so there was a lot of emphasis on interacting with databases through a .NET application and using the tools of .NET such as NuGet. I found that after working in .NET and using the tools it offered, I strongly preferred C# over Java for OOP.

Library Database

The first project I would like to show is the ORM (Object Relational Mapper) lab in my CSCI-2910 class. In this project, we had to interact with a SQLite database to perform all CRUD operations with objects that I had made classes for in a .NET application. This was supposed to model a library with books and rentals, so it had objects for a book, author, etc. We also had to make sure to use generics so that any object could be used with the Querybuilder class we created to perform each CRUD operation. This was done with the generic types in C# along with using an interface to get the Id of each entity.


What was used for this lab:

  • C#, .NET 6.0
  • NuGet for SQLite packages
  • SQLite

Here is a link to the gitub repository for the lab: SQLite Database Querybuilder

JSON Lab

The next project from this class was a JSON and Postman lab where we had to interact with a Google API for the Google books library with Postman. Then after we receive the JSON from interacting with the Google API we had to interact with this JSON by parsing it into C# objects and vice versa. We had to do this with both the JSON converter included in the system libraries with .NET and the Newtonsoft package from NuGet. This was a good lab to show how you can interact with many different tools to get the results you need.


What was used for this lab:

  • C#, .NET 6.0
  • NuGet for Newtonsoft
  • JSON
  • Postman
  • Google Books API
